HOUSECHECK — EIRCODE, FULL DESCRIPTION AND PHOTO GALLERY UPDATE

This update assumes migrations 001 and 002 are already installed.

1. Stop HouseCheck in VS Code with Ctrl+C.
2. Open this ZIP and copy all files and folders into:
   C:\Users\Chris\Desktop\housecheck
3. Choose "Replace the files in the destination" when Windows asks.
4. The ZIP does not contain .env.local or package-lock.json, so it will not
   replace your private keys or your corrected npm lock file.
5. In Supabase open SQL Editor -> New query.
6. Copy and run:
   supabase\migrations\003_property_source_content.sql
7. Restart HouseCheck:
   npm run dev
8. Open:
   http://localhost:3000/review
9. Review each property:
   - check or enter the Eircode;
   - verify the full source-listing description;
   - upload the actual house and land photographs;
   - keep the red-boundary image in Boundary evidence (it stays private);
   - tick the permission-to-display checkbox;
   - press Approve and publish.

WHAT CHANGES

- Eircode appears on the property card, detail page and comparison page.
- The full approved listing description appears with paragraph breaks and
  source attribution.
- Each property can have up to 30 JPG, PNG or WebP photographs in a gallery.
- New or changed source descriptions/photos return to Pending for review.
- Public pages hide unapproved source descriptions and photos.
- Existing approved rows return to Pending once after migration 003 so you can
  verify their content.
- Eircodes already present in an address are backfilled automatically.

The Daft importer remains permission-gated. Manual approval controls what is
published in HouseCheck; it does not grant rights to copy source content.
