HOUSECHECK REVIEW APPROVAL UPDATE

1. Stop the running website with Ctrl+C.
2. Copy all files and folders from this update into:
   C:\Users\Chris\Desktop\housecheck
3. Choose Replace the files in the destination when Windows asks.
4. Do not delete or replace your .env.local file. This update does not contain one.
5. In Supabase SQL Editor, run:
   supabase\migrations\002_review_approval.sql
6. Restart the website:
   npm run dev
7. Open:
   http://localhost:3000/review
8. Enter your existing INGEST_API_KEY from .env.local.
   You can instead add REVIEW_ADMIN_KEY=another-private-password and use that.
9. Upload the aerial boundary image, correct the setting tags, and approve the house.

Classification used by this update:
- Woodland on property: woodland is inside the red property boundary.
- Forest edge: the boundary touches forest outside the property.
- Near forest: forest is nearby but separate.
- River frontage: the property boundary directly meets a river.
- Riverside: near a river, but frontage is not confirmed.

New imports stay pending and hidden until approved.
